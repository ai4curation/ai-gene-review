# BioReason-Pro comparison — stats notebooks

Reproducible Jupyter notebooks that recompute the summary statistics quoted in
[`../../BIOREASON_COMPARISON.md`](../../BIOREASON_COMPARISON.md) and the
[manuscript](../article/manuscript.md) **directly from the committed per-gene
files** under `genes/`. Nothing is hard-coded — every table and figure is derived
on the fly, so the notebooks double as an audit of the paper's numbers.

## Notebooks

| Notebook | Reproduces | Source files |
|----------|------------|--------------|
| [`01_narrative_scores.ipynb`](01_narrative_scores.ipynb) | Overall means (RL 3.7/2.9, SFT 2.9/2.7), **Table 1** (score distribution), **Table 2** (per-organism means), top performers / critical failures, the per-organism figure | `genes/*/*/*-bioreason-{rl,sft}-review.md` |
| [`02_prediction_assessments.ipynb`](02_prediction_assessments.ipynb) | Per-term assessment distribution (de Crécy-Lagard taxonomy; **Table 5** analog) and terms-per-gene stats | `genes/*/*/*-gogpt-leaf-predictions.yaml` |

Shared parsing lives in [`bioreason_stats.py`](bioreason_stats.py); the notebook
cell contents are generated by [`build_notebooks.py`](build_notebooks.py) so they
are easy to review in a diff and stay in sync with the module.

> **Note on notebook 02.** On `main` the prediction YAMLs are raw web-export stubs
> (every term `UNC`). The manually-verified Table 5 distribution (≈67% CNN,
> 11.5% NPI, 4.5% COR) was produced on the `feat/bioreason-hf-catalogue` working
> branch. The notebook computes the distribution from whatever is on disk rather
> than hard-coding Table 5, so it shows the current state here and reproduces
> Table 5 verbatim when run on the review branch.

## Running

Dependencies are pinned with [`uv`](https://docs.astral.sh/uv/) (`pyproject.toml`
+ `uv.lock`).

```bash
cd projects/BIOREASON_COMPARISON/notebooks

# regenerate the .ipynb files from the builder and execute them in place
uv run python build_notebooks.py
uv run jupyter nbconvert --to notebook --execute --inplace *.ipynb

# or work interactively
uv run jupyter lab
```

Regenerated figures are written to `figures/*.repro.png` (kept out of git) and
should match the committed `../article/figures/` versions.
