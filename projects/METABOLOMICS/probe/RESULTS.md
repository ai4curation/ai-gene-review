# Metabolomics → GO Bridge: protonation-normalization coverage probe

Generated by [`coverage_probe.py`](coverage_probe.py) — all numbers computed
live from OLS4 (ChEBI), the Rhea REST API, and the GO `rhea2go` mapping.
Re-run with `uv run python coverage_probe.py --write-results`.

## Why this probe exists

Metabolomics repositories report metabolites as their **neutral** species;
Rhea writes reaction participants in their **major protonation state at**
**pH 7.3**. Matching the reported ChEBI id directly against Rhea therefore
misses the bridge. We expand each metabolite over ChEBI's
`is_protonated_form_of` / `is_deprotonated_form_of` relations (its
*protonation family*) and re-check. The EXACT vs NORMALIZED gap is the
value added by protonation normalization.

## Residual misses (a *different* mismatch class)

These resolved to ChEBI but matched no Rhea reaction even after
protonation normalization. They expose a second ID-mismatch class that
protonation expansion does **not** fix — **stereochemistry / anomer** and
**generic-vs-structurally-specific** ChEBI mismatch (e.g. Rhea uses a
stereospecific or anomer-specific form, or a structurally-defined child of
a generic 2D parent). Fixing these needs tautomer/enantiomer/`has parent
hydride` traversal or structure-skeleton (InChIKey) matching — a follow-up.
