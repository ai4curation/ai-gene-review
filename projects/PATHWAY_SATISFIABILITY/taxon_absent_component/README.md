# Taxon-absent-component detector

A small, runnable prototype that turns one recurring IBA over-annotation pattern
into an automatable screen. It is the **signaling-domain, genome-content**
special case of the [Pathway satisfiability](../../PATHWAY_SATISFIABILITY.md)
engine (which resolves *metabolic* modules against an expression oracle).

[parent project ← Pathway satisfiability](../../PATHWAY_SATISFIABILITY.md) ·
[origin ← IBA Annotation Quality](../../IBA_REVIEW.md)

## The pattern

Some process/pathway GO terms **entail an obligate component**. `GO:0007259`
"signaling via **JAK**-STAT" entails a Janus kinase. Read as a boolean formula
over required components under a *genome-content oracle*, the annotation is
**unsatisfiable** in a genome that encodes no such component — so an IBA that
transferred it (from metazoan seeds, say) is a candidate
`LINEAGE_OR_TAXON_MISMATCH` over-propagation.

This came out of the *Dictyostelium* development review: Dd-STATa/c were
annotated JAK-STAT, but *Dictyostelium* has **no JAK** (the STATs are activated
by the TKL kinases Pyk2/Pyk3), so the term was rescoped to the JAK-independent
parent `GO:0097696`.

## What it does

`detect.py` reads `signatures.yaml` — each case maps a GO term to its obligate
component and one or more **InterPro signatures** — and, for each, counts
UniProtKB members of the signature in the **target** taxon (default
*D. discoideum*, 44689) and a **control** taxon where the component is known
present (default human, 9606). It classifies:

| Status | Meaning |
|--------|---------|
| `COMPONENT_PRESENT` | target encodes a member → satisfiable w.r.t. this component; not flagged |
| `ABSENT_CANDIDATE` | target 0, control >0 → **candidate** unsatisfiable; must be corroborated |
| `INCONCLUSIVE` | control 0 (bad signature) or an API error → no call |

Counts are fetched **live** from UniProt REST and never hard-coded; a failed
fetch yields `INCONCLUSIVE`, not a made-up number.

## Run it

```bash
cd projects/PATHWAY_SATISFIABILITY/taxon_absent_component
uv run python detect.py --write RESULTS.md          # default DICDI vs human
uv run python detect.py --target 44689 --control 9606
```

Current output is in [RESULTS.md](RESULTS.md) (regenerated by the command above).

## The critical caveat — this is a TRIAGE screen, not an oracle

**Signature-absence ≠ genome-absence for divergent lineages.** The config
deliberately includes two false-positive controls, and the live run reproduces
them:

- **STAT itself** (`IPR013801`): control 100, target **0** — yet *Dictyostelium*
  demonstrably HAS functional STATs. They are too sequence-divergent to carry
  the metazoan STAT InterPro signature.
- **P2X receptor** (`IPR001429`): control 26, target **0** — yet *Dictyostelium*
  encodes divergent P2X receptors (Fountain et al. 2007, *Nature* 448:200).

Both come back `ABSENT_CANDIDATE`, identical to the true JAK absence. The raw
count **cannot tell true absence from divergent-present**. What separates them is
the `corroboration` field — independent evidence (here: the documented
JAK-independence of *Dictyostelium* STAT activation makes JAK a real absence,
whereas the STAT/P2X literature makes those false positives).

## Honest conclusion

- **JAK / JAK-STAT is the one corroborated true positive** in this run: absent by
  signature **and** independently known to be absent → the `GO:0007259`→
  `GO:0097696` rescope is sound.
- The metabotropic **P2Y (GPCR) purinergic** case is a *candidate* only; the P2X
  control shows why "no purinergic receptors in *Dictyostelium*" would be an
  overstatement — *Dictyostelium* has ionotropic P2X, just not (as far as the
  signature shows) the metabotropic P2Y GPCR that `GO:0035589` requires.
- **Use as a pre-filter, not a verdict.** The detector cheaply nominates IBA
  process/pathway rows for taxon-mismatch review; a curator (or a downstream
  divergence-aware search — HMM/structure, not just InterPro) confirms whether
  the absence is real before any `REMOVE`.

## Possible extensions

- Divergence-robust presence test (profile-HMM or structure search) to cut the
  false-positive rate in distant lineages.
- Drive the target taxon and candidate terms straight from a review's
  `LINEAGE_OR_TAXON_MISMATCH` `propagation_review` rows, closing the loop with
  the [IBA project](../../IBA_REVIEW.md).
- Broaden component signatures from single subfamilies to full families (e.g. a
  complete P2Y panel) so a single-subfamily miss doesn't skew the call.
