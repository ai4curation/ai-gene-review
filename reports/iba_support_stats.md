# IBA independent-literature support — summary stats

Generated by `scripts/analyze_iba_support.py` (per-annotation detail in
`reports/iba_support_stats.tsv`). This is the **first-step baseline** before we
run the `asta` deep-research client to find additional support for unsupported
IBAs.

## What was measured

Every `genes/*/*/*-ai-review.yaml` was scanned for annotations with
`evidence_type: IBA` (6,618 of them; ~all from `GO_REF:0000033`, the PANTHER
IBA reference). An IBA counts as **independently supported by the literature**
if it carries at least one real PMID/DOI in `supported_by` or
`review.additional_reference_ids` — i.e. a citation *other than* the PANTHER
GO_REF the IBA was propagated from. These independent PMIDs are a mix of
pre-existing GOA annotations on the same gene and ones we added during review.

For supported IBAs, each cited paper was matched against the
`publications/` cache to find the **cheapest source that confirms the claim**:

- **abstract** — the verbatim `supporting_text` is found inside the paper's
  abstract, or the cached paper is abstract-only (`full_text_available: false`),
  so an abstract is sufficient.
- **review** — the cited paper is a review article.
- **primary full-text** — the supporting quote is only found in the full body
  of a primary paper.

## Headline numbers

| Metric | Count | % of IBAs |
|---|---:|---:|
| Total IBA annotations | 6,618 | 100% |
| **With independent literature support** | **3,304** | **49.9%** |
| Without independent support | 3,314 | 50.1% |

### Breakdown of the 3,304 supported IBAs (cheapest sufficient source)

| Source tier | Count | % of supported |
|---|---:|---:|
| Abstract suffices | 2,852 | 86.3% |
| Primary full-text only | 373 | 11.3% |
| Support present but quote not localizable* | 79 | 2.4% |
| Review article | 0 | 0.0% — **not measurable, see below** |

\* mostly `additional_reference_ids` carrying no `supporting_text` quote to
localize.

## The "review" breakdown cannot be answered yet

The abstract split is robust, but **which IBAs can be confirmed from a review
article is currently unanswerable** because review-vs-primary classification is
missing from almost the entire publications cache:

- Only **737 of 21,091** cached publications (3.5%) carry any PubMed
  publication-type metadata; only **14** are tagged `Review`.
- Of the **5,334** literature references cited as IBA support, only **209
  (3.9%)** resolve to a cached paper that has pub-type metadata at all.
- Only **90 (2.7%)** of supported IBAs cite *any* paper for which we could even
  attempt a review/primary determination.

So the "0 reviews" above is an artifact of missing metadata, not a real finding.
**Prerequisite:** backfill `pubmed_publication_types` into the publications
cache before the review-vs-primary split is meaningful. No new code is needed —
`src/ai_gene_review/etl/publication.py` already fetches `pubmed_publication_types`
and derives `publication_type`, and `just refresh-publications-force-all`
re-fetches the whole cache. The catch is operational: it is a bulk re-fetch of
~21k PubMed records (rate-limited, ~hours, needs NCBI network access), not an
instant step. Once backfilled, re-running `scripts/analyze_iba_support.py` will
populate the review tier.

## Takeaways for the next step (asta deep research)

- ~50% of IBAs (3,314) have **no** independent literature support today — these
  are the primary targets for the `asta` deep-research support-finding pass.
- Where support *does* exist, it is overwhelmingly **abstract-level** (86%),
  which suggests an abstract-first confirmation loop will catch most cases
  cheaply; reserve full-text fetches for the ~11% that need them.
- Expect false positives from `asta`; the per-annotation TSV gives an agent the
  exact gene/term/existing-refs context needed to sift results.

### Running the asta support-finding pass

A support-finding target now exists, modeled on the openscientist
hypothesis-confirmation workflow. For one gene it researches each IBA annotation
that lacks independent support and writes a per-annotation report an agent can
sift:

```bash
just gene-iba-support-list human CFAP300                 # candidate IBAs (unsupported by default)
just gene-iba-support-research asta human CFAP300 --dry-run   # preview the commands
just gene-iba-support-research asta human CFAP300             # run; outputs under
#   genes/human/CFAP300/CFAP300-hypotheses/iba-support-<term>/asta.md
```

Use `--include-supported` to also re-confirm already-supported IBAs,
`--annotation-term-id GO:nnnnnnn` to target a single annotation, and
`--overwrite` to replace existing outputs. The bare `asta` provider is also
available for whole-gene deep research via `just deep-research-asta`.
